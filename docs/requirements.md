# TelopMaker 要件定義書

**文書バージョン**: 1.0  
**作成日**: 2026-04-05  
**ステータス**: ドラフト

---

## 1. プロジェクト概要

### 1.1 目的
MP4動画を入力として受け取り、話者の音声を自動で文字起こしし、テロップ（字幕）を生成・動画に合成するアプリケーション「TelopMaker」を開発する。

### 1.2 背景・動機
動画編集において、テロップ付けは最も工数のかかる作業の一つである。音声認識技術を活用し、この作業を自動化することで編集コストを大幅に削減する。

### 1.3 スコープ

| 区分 | 内容 |
|------|------|
| フェーズ1（今回） | 音声認識による自動文字起こし + テロップ生成・合成 |
| フェーズ2（将来） | 無音部分の自動カット |
| フェーズ3（将来） | BGM・SE追加機能 |

## 2. 機能要件

### 2.1 コア機能（フェーズ1）

#### F-001: 動画読み込み
- MP4形式の動画ファイルを入力として受け付ける
- ファイル選択UIまたはドラッグ&ドロップで読み込む
- 動画のメタ情報（解像度、長さ、コーデック）を表示する

#### F-002: 音声抽出
- MP4から音声トラックを抽出し、16kHz・モノラルWAV形式に変換する
- FFmpegを使用して変換処理を行う

#### F-003: 音声認識・文字起こし
- 抽出した音声データに対してAI音声認識を実行する
- 日本語の音声を高精度でテキストに変換する
- 各セグメントごとに開始時刻・終了時刻（タイムスタンプ）を取得する
- 認識言語は日本語をデフォルトとし、将来的に他言語も対応可能な設計とする

#### F-004: SRTファイル生成
- 文字起こし結果をSRT（SubRip Subtitle）形式の字幕ファイルとして出力する
- SRTファイルはUTF-8エンコーディングで生成する
- タイムコード形式: `HH:MM:SS,mmm --> HH:MM:SS,mmm`
- SRTファイルは副産物として保存し、他の動画編集ソフトでも再利用可能とする

#### F-005: テロップの動画合成（ハードサブ）
- 生成したSRTファイルを元に、テロップを動画に焼き込む
- FFmpegのsubtitlesフィルタ（libass）を使用して合成する
- 出力形式はMP4とする

#### F-006: テロップ表示位置の選択
- テロップの表示位置をユーザーが選択できる
- 選択肢: 下部中央（デフォルト）、上部中央、中央、任意のY座標指定
- ASS形式のスタイル指定により位置制御を実現する

#### F-007: テロップスタイル設定
- 以下のスタイルパラメータをユーザーが設定できる:
  - フォント種類（デフォルト: Noto Sans CJK JP Bold）
  - フォントサイズ
  - 文字色
  - アウトライン（縁取り）の色と太さ
  - 背景ボックスの有無と色

#### F-008: テロップ文字の手動編集
- 音声認識結果を動画出力前にユーザーが確認・編集できる
- 誤認識箇所の修正、不要セグメントの削除が可能

#### F-009: 処理進捗の表示
- 各処理ステップ（音声抽出→認識→SRT生成→合成）の進捗をUIに表示する
- 推定残り時間または進捗率を表示する

### 2.2 将来機能（フェーズ2以降）

| ID | 機能 | フェーズ |
|----|------|----------|
| F-101 | 無音区間の自動検出・カット | 2 |
| F-102 | BGM追加（テンプレートまたはファイル指定） | 3 |
| F-103 | SE（効果音）追加 | 3 |
| F-104 | 複数話者の識別（話者分離） | 未定 |
| F-105 | リアルタイムプレビュー | 未定 |

## 3. 非機能要件

### 3.1 パフォーマンス
- 10分の動画を5分以内に処理完了すること（M1 Mac以降を想定）
- CPU環境（GPU非搭載マシン）でも動作すること（処理時間の制約は緩和可）

### 3.2 コスト
- ランニングコストゼロ（外部API課金なし）
- すべての処理をローカルで完結させる
- 使用するライブラリ・モデルはすべてオープンソースまたは無料

### 3.3 対応環境
- macOS（Apple Silicon: M1/M2/M3/M4）を主要ターゲットとする
- Python 3.10以上
- 将来的にWindows/Linux対応も視野に入れる

### 3.4 入出力仕様

| 項目 | 仕様 |
|------|------|
| 入力形式 | MP4（H.264/H.265） |
| 出力形式 | MP4（H.264）+ SRTファイル |
| 最大ファイルサイズ | 制限なし（ストレージ依存） |
| 対応解像度 | 入力動画の解像度を維持 |
| 音声チャンネル | ステレオ/モノラル対応 |

### 3.5 ユーザビリティ
- 技術者でなくても操作できるUI
- 最小操作: ファイル選択 → 実行ボタン → 出力ファイル取得（3ステップ）

## 4. 技術選定

### 4.1 音声認識エンジン

**採用: faster-whisper（large-v3-turboモデル, INT8量子化）**

| 選択肢 | 日本語精度 | 速度 | コスト | ローカル実行 | 評価 |
|--------|-----------|------|--------|-------------|------|
| **faster-whisper** | 高 | 高速（Whisper比4倍） | 無料 | 可 | **採用** |
| Whisper（本家） | 高 | 基準 | 無料 | 可 | 次点 |
| WhisperX | 最高（forced alignment） | 高速 | 無料 | 可 | 将来検討 |
| Google STT | 高 | - | 60分/月無料 | 不可 | 不採用 |
| Vosk | 中 | 高速 | 無料 | 可 | 精度不足 |

**採用理由**:
- Whisperと同等の精度を維持しつつ約4倍の速度改善
- INT8量子化により、GPUなしのCPU環境でもVRAM/メモリ使用量を約50%削減
- VADフィルタにより無音区間の誤認識（ハルシネーション）を抑制
- large-v3-turboモデルでlarge-v3に近い精度を維持しつつ処理速度を大幅改善
- 完全ローカル実行でランニングコストゼロ

### 4.2 動画処理

**採用: FFmpeg + ffmpeg-python**

| 選択肢 | 日本語フォント | 処理速度 | スタイル自由度 | 評価 |
|--------|--------------|---------|--------------|------|
| **FFmpeg + ffmpeg-python** | 対応（libass） | 最速 | 高（ASS形式） | **採用** |
| MoviePy | 条件付き | 低速 | 中 | 不採用 |
| OpenCV + Pillow | 完全対応 | 低速 | 最高 | 不採用 |

**採用理由**:
- 音声抽出とテロップ焼き込みの両方を1ツールで処理可能
- C実装のため処理速度が圧倒的に速い
- libassによるASS形式でフォント・位置・色・アウトラインの制御が可能
- SRTからのハードサブが最もシンプルかつ高品質

### 4.3 UI/アプリ形態

**採用: Streamlit（フェーズ1）**

| 選択肢 | 開発コスト | UX | 配布 | 評価 |
|--------|----------|-----|------|------|
| **Streamlit** | 小 | 高（ブラウザUI） | pip install | **採用（フェーズ1）** |
| PyQt6 | 中〜大 | 高 | PyInstaller | 将来検討 |
| Python CLI | 最小 | 低 | pip install | 不採用 |

**採用理由**:
- ファイルアップロード、進捗バー、プレビュー再生が少ないコードで実装可能
- ブラウザベースでOS非依存
- 個人・社内ツールとして最速で開発できる
- 将来的に配布が必要になった場合はPyQt6への移行を検討

### 4.4 技術スタック全体像

```
Python 3.10+
├── faster-whisper     ... 音声認識（large-v3-turbo, INT8）
├── ffmpeg-python      ... 音声抽出・テロップ焼き込み
├── streamlit          ... Web UI
└── FFmpeg（システム）  ... 動画処理エンジン（要インストール）

フォント: Noto Sans CJK JP（Google提供、無料）
```

## 5. 処理フロー

```
[ユーザー]
  │ MP4ファイルをアップロード
  ▼
[F-001] 動画読み込み・メタ情報取得
  │
  ▼
[F-002] FFmpegで音声抽出 → 16kHz モノラルWAV
  │
  ▼
[F-003] faster-whisperで音声認識
  │ → セグメントリスト（テキスト + 開始時刻 + 終了時刻）
  ▼
[F-008] 認識結果の確認・手動編集（UI上）
  │
  ▼
[F-004] SRTファイル生成・保存
  │
  ▼
[F-006/007] テロップ位置・スタイル設定（UI上）
  │
  ▼
[F-005] FFmpegでテロップ焼き込み → MP4出力
  │
  ▼
[ユーザー]
  ダウンロード: テロップ付きMP4 + SRTファイル
```

## 6. 画面構成（Streamlit）

### 6.1 メイン画面
1. **ファイルアップロードエリア** - MP4ファイルの選択/ドラッグ&ドロップ
2. **動画情報表示** - 解像度、長さ、ファイルサイズ
3. **実行ボタン** - 文字起こし処理を開始
4. **進捗バー** - 処理ステップと進捗率

### 6.2 編集画面（文字起こし結果表示後）
1. **セグメント一覧** - タイムスタンプ付きテキストの一覧表示・編集
2. **テロップ設定パネル** - 位置、フォント、サイズ、色、アウトラインの設定
3. **合成実行ボタン** - テロップ焼き込みを開始
4. **ダウンロードボタン** - MP4 / SRTファイルのダウンロード

## 7. 依存関係・前提条件

### 7.1 システム要件
- Python 3.10以上
- FFmpeg（システムにインストール済みであること）
- Noto Sans CJK JPフォント（テロップ表示用）
- 推奨RAM: 8GB以上（faster-whisper INT8量子化使用時）

### 7.2 Pythonパッケージ（主要）
- `faster-whisper` - 音声認識
- `ffmpeg-python` - FFmpeg操作
- `streamlit` - Web UI

### 7.3 初回セットアップ
- faster-whisperモデル（large-v3-turbo）の初回ダウンロード（約1.5GB）
- FFmpegのインストール（`brew install ffmpeg` 等）
- 日本語フォントのインストール

## 8. リスク・課題

| リスク | 影響 | 対策 |
|--------|------|------|
| 日本語の固有名詞・専門用語の認識精度 | テロップの誤字 | 手動編集機能(F-008)で対応。将来的にfine-tuningを検討 |
| 複数話者の重複発話 | 認識精度低下 | フェーズ1では単一話者を想定。話者分離は将来対応 |
| CPU環境での処理速度 | 長い動画で待ち時間大 | INT8量子化で軽量化。進捗表示(F-009)でUX改善 |
| FFmpegのクロスプラットフォーム差異 | 環境依存バグ | macOS優先。他OSは段階的にテスト |
| Streamlitのファイルサイズ制限 | 大容量動画の読み込み失敗 | `server.maxUploadSize`設定で上限引き上げ |

---

## 9. 制約事項

- フェーズ1はオフライン（事後処理）のみ対応。リアルタイム字幕は対象外
- 入力形式はMP4のみ（他形式は将来対応）
- 音声認識は日本語のみ（他言語は将来対応）
- テロップスタイルはSRT/ASS形式で表現可能な範囲に限定

## 10. 用語定義

| 用語 | 定義 |
|------|------|
| テロップ | 動画上に表示されるテキスト（字幕） |
| ハードサブ | 字幕を動画映像に直接焼き込むこと（取り外し不可） |
| SRT | SubRip Subtitle形式。字幕の業界標準フォーマット |
| ASS | Advanced SubStation Alpha形式。SRTより高度なスタイル制御が可能な字幕フォーマット |
| VAD | Voice Activity Detection。音声区間検出。無音部分を判別する技術 |
| セグメント | 音声認識における1つの発話区間（テキスト + 開始/終了時刻） |
